Built to showcase practical scheduling skills applied across real government ICT delivery environments. All project details are fictional and constructed for portfolio purposes. Phase Summary
Phase 1 — Project Initiation (10 days, 2–15 Jan)
LOA received, Project Charter drafted and approved, stakeholder register, communication plan, first status report distributed.
Phase 2 — Requirements and Planning (14 days, 16 Jan–4 Feb)
BRS sessions conducted, BRS and SRS drafted and signed off, master project schedule developed and baselined, risk register established.
Phase 3 — System Development and Integration (22 days, 5 Feb–6 Mar)
Four concurrent workstreams running in parallel.
Workstream A — Portal and User Management: architecture design, module development, internal testing.
Workstream B — Data Migration: extraction, cleansing, dry run, sign-off.
Workstream C — System Integration: environment setup, integration testing Phase 1 and 2, sign-off.
Workstream D — Infrastructure and Deployment Planning: server provisioning, environment configuration, deployment checklist and readiness confirmation.
Phase 4 — Testing and Acceptance (25 days, 7 Mar–10 Apr)
UAT plan prepared, UAT conducted with client stakeholders, defects logged and resolved, UAT sign-off, FAT conducted, defects resolved, FAT sign-off, Go-Live readiness checklist completed.
Phase 5 — Deployment and Close-Out (8 days, 10–22 Apr)
Go-Live executed, post-deployment monitoring, handover documentation, client training, close-out report submitted, project formally closed.
